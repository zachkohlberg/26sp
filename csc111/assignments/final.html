CSC 111 Data Structures and Abstractions

A4: Final Project

This project is meant to be completed during class. Our last four classes will cover each part of the assignment and allow time to work on it. You should be able to complete most of it during these class meetings.

Description

We will implement a markov text generation algorithm. This program analyzes which words tend to follow other words in one or more sample texts, and then it uses that information to generate new text that resembles the sample text. This is superficially similar to an LLM, but far less sophisticated and relatively easy to implement.

This project would involve implementing a hash table data structure to track which words follow other words in the sample text.

Tasks

These tasks will not go into as much detail as other assignments. I’m just going to specify the requirements for each task, and we’ll cover the details during our class meetings. Refer to your notes from class and the example code in the class repo for additional guidance. If you miss a class, then you should meet with a classmate to review their notes and review the material in the class repo.

Task 1 (10 points): Hash Table and Hash Function

Implement a hash table class as described below:

  • generic: HashTable<K, V>, where K represents the key type and V represents the value type
  • constructor: HashTable(Hash<K>, size)
    • accept a hash function through the Hash<K> functional interface created during class
    • accept a size for the hash table’s internal array, which will affect its overall efficiency and the amount of memory it requires
  • instance variables for the hash function Hash<K> and data array KeyValueNode<K, V>[]
  • a get(K): V to retrieve values from the table
  • a set(K, V) method to add/update values in the table

Also implement at least one hash function for strings. You may want to try several different hash functions and compare them to see which produces the fastest hash table, but this is optional.

Task 2 (5 points): Read Corpus

Write a program named Markov.java that accepts one or more filenames through its command line args, reads each file (assume all are plaintext), and processes them as described below. This should result in two data structures:

  • a list of all the tokens that appear at the beginning of a “section” (see below)
  • a hash table mapping every token found in the input files to a list containing
    • every token that immediately followed it in any of the input files (if the same token followed it more than once, then you should include duplicates in the list)
    • null for every time this token appeared at the end of a “section”

The definition of a section is flexible. You can choose a different definition, but in my examples I will treat a paragraph (or anything similar to a paragraph) as a section of text.

Processing a text file:

  • use the .split method and an appropriate regular expression to split the file’s contents into sections
  • for each section
    • use the .split method and a regular expression to split the section into tokens
    • add the first token to the list of starting tokens
    • for each token except the last token
      • if it doesn’t already appear as a key in the table, add a new entry with the token as a key and a new empty list as its value
      • retrieve the token’s list from the table
      • add the next token in the section to the list
    • for the last token in the section
      • if it doesn’t already appear as a key in the table, add a new entry with the token as a key and a new empty list as its value
      • retrieve the token’s list from the table
      • add null to the list

For example, the following text (each line counting as its own section) should produce the mapping shown after it.

#################
# Text Sections #
#################

Hello, my name is Bob.

Hello Bob, my favorite color is blue.

Hello, my favorite color is also blue.

##################
# Starter tokens #
##################

- "Hello,"
- "Hello"
- "Hello,"

#######
# Map #
#######

- "Hello,"   -> "my", "my"
- "my"       -> "name", "favorite", "favorite"
- "name"     -> "is"
- "is"       -> "Bob.", "blue.", "also"
- "Bob."     -> null
- "Hello"    -> "Bob,"
- "Bob,"     -> "my"
- "favorite" -> "color", "color"
- "color"    -> "is", "is"
- "blue."    -> null, null
- "also"     -> "blue."

Task 3 (5 points): Generate Text

Implement the text generation algorithm outlined during class. Add the ability for your Markov program to generate one or more sections of text after it processes all of its input files. The details are up to you: generate a set number of sections, allow the user to interactively generate sections until they quit the program, or something else.

Optional: if you want to improve the quality of the generated text, then you could modify your hash table to map sequences of 2 or 3 tokens to a list of next tokens. This will require some changes to the previous part of your program where you construct the table, but it can result in significantly more interesting and coherent output.

Task 4 (5 points): Demo

Gather some input (either public domain material or something you made yourself) to serve as a corpus and use it to generate some text. Try different material and settings until your program creates something you find interesting or amusing. Include the corpus and output with your submission.
