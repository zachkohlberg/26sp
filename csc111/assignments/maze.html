Tasks

1. Maze Class (5 points)

Tip

You will have implemented this exact class or something close to it in the lab and are free to use that code in this assignment

Write a Maze class (Maze.java). This class must have the variables, constructors, and methods described below. You should not have any other instance variables or static variables.

  • instance variables (private)
    • data: char[][]
    • width: int
    • height: int
  • constructor
    • Maze(w: int, h: int)
      • initialize a new maze with width w and height h
      • initialize every character in the maze
      • assume that w and h are positive
    • Maze(d: char[][])
      • initialize a new maze with width d.length and height d[0].length
      • initialize data as a copy of d
        • you must copy the characters from d using loops, not by calling Arrays.copy or a similar method that would do it for you
      • assume d has a positive length
      • assume every element of d has a positive length and that all elements have the same length
    • Maze(f: File) throws FileNotFoundException
      • initialize the maze with the information read from f
      • assume f is formatted exactly as described in the maze file format explained below
  • methods
    • inBounds(x: int, y: int): boolean
      • returns true if (x, y) represents a point within the maze and false if it does not
    • get(x: int, y: int): char
      • returns the character at (x, y)
      • assume (x, y) is in bounds
    • set(x: int, y: int, c: char): void
      • sets the character at (x, y) to c
      • assume (x, y) is in bounds
    • getWidth(): int
    • getHeight(): int
    • toString(): String
      • return a string representation of the maze
      • the maze data must be arranged so that the first (upper-left) character in the string is data[0][0] and the last (lower-right) character in the string is data[width - 1][height - 1] (in other words: the y-axis is flipped compared to a standard Cartesian coordinate system)

Tip

This class uses x and y coordinates to index into the maze rather than row and column coordinates. To avoid confusion, you should stick to x and y coordinates in your code, treating the array’s first index as the x coordinate and the second index as the y coordinate.

Maze File Format

A properly-formatted maze file:

  • first line: the width and height separated by a space
  • maze: a number of lines equal to the height, each containing a number of characters equal to the width
  • comments/extra: may have additional lines of text after the maze; these should be ignored when reading the file

Example maze file (save as maze.txt for the maze test program to find it):

19 5
# # ### #   #   ###
# # #   #   #   # #
### ### #   #   # #
# # #   #   #   # #
# # ### ### ### ###

This is extra text in the comments section. Your code shouldn't read this part,
or at least shouldn't use it when initializing a Maze object.

Also, nothing in the format says the maze file has to be an actual solvable
maze. This maze is just the word HELLO so you can tell that your maze read the
file correctly and that its toString method works correctly.

Task 3: Recursive Solve Method (10 points)

Add the MazeConstants.java file to your project before writing the solve method:

public class MazeConstants {
    public static final char EMPTY = ' ';
    public static final char PATH = '.';
    public static final char START = 'S';
    public static final char VISITED = 'v';
    public static final char WALL = '#';
}

We must obey the following rules when trying to solve the maze:

  • positions containing the space character ' ' are considered empty
  • positions containing any other character are considered occupied, and we cannot enter them
  • when we begin solving the maze, we will start at an empty position within the bounds of the maze
  • from our current position, we are allowed to move to the adjacent north, south, east, or west position as long as it is empty
    • for example, we could move from (7, 3) to (7, 4), which is one cell south, but not to (7, 5) (two cells south) or (8, 4) (one cell diagonally southeast)
  • our goal is to find a path that leads outside of the maze (in other words, to reach a position that is out of bounds)

You will use the following recursive algorithm to solve the maze:

  • input
    • a maze to solve and a starting position (x, y)
  • first base case
    • condition: our starting position is out of bounds
    • result: we’ve solved the maze (out of bounds means we’ve exited the maze)
  • second base case
    • condition: our starting position isn’t empty (doesn’t contain MazeConstants.EMPTY)
      • you can use the maze’s get method to check whether a position is empty: m.get(12, 27) == MazeConstants.EMPTY
    • result: we’ve failed to solve the maze (we’re trying to move through an occupied cell, which isn’t allowed)
  • recursive case
    • mark our starting position as MazeConstants.VISITED
      • you can use the maze’s set method to mark a position: m.set(MazeConstants.VISITED, 5, 11);
    • for each position we are allowed to move to, attempt to solve the maze starting at that position
      • these positions are (x - 1, y), (x + 1, y), (x, y - 1), and (x, y + 1)
    • if we succeed in solving the maze, then mark our starting position as MazeConstants.PATH

You must implement this algorithm as a recursive static method:

  • name: solveMaze
  • return type: boolean (true if solved and false if not)
  • parameters:
    • m: Maze is the maze to solve
    • x: int is the starting position’s x-coordinate
    • y: int is the starting position’s y-coordinate

Finally, update your main method to use solve maze:

  • after printing the maze, solve it
  • after solving the maze, mark the starting position as MazeConstants.START
  • print a message telling the user whether the maze was solved
  • print the maze again to show the solution path (if one was found) and all the visited positions

Requirements

  • solveMaze implements the recursive algorithm described above
  • solveMaze marks a path to an exit if it solves the maze
  • solveMaze correctly returns whether or not it solved the maze
  • main calls your solver method, marks the start position, prints the maze, and prints whether it was solved
