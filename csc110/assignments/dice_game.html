A2: Dice Game

This program allows the user to play a dice game similar to Yahtzee. Our game will include most of the scoring options that Yahtzee uses, but we’ve left out a few of the trickier ones. This is a single player game, so the player will just be trying to earn the highest possible score.

This project involves the following topics:

  • Breaking a larger program into many small functions
  • Generating random numbers (you’ll implement your own LCG like the one we showed in class)
  • Arrays
  • While loops

Tasks

1. Dice-Rolling Functions (5 points)

Create a new program called DiceGame.java. To start, we’re going to write some functions that will allow you to simulate rolling dice.

Add an int variable named seed to your program. This is a “global” variable, and it must be placed outside of any functions. Only two of your functions will use it directly, and it lets us keep track of the state of our random number generator. You are not allowed to have any other global variables in your program except for the category names suggested in task 3. All other variables must be function parameters or local variables declared within functions.

rollDie

  • generates a random number from 1 to 6, simulating a die roll
  • parameters: none
  • return type: integer

This is similar to the PRNG example we wrote during class, but with a few changes:

  • use the same formula to update the seed: seed = (seed * 9301 + 47297) % 233280
  • to scale this integer to be a random number from 0 to 5, multiply by 6 and divide by 233280: seed * 6 / 233280
  • add 1 to make the range 1 to 6 instead of 0 to 5: (seed * 6 / 233280) + 1
  • remember to return the number from 1 to 6

rollAll

  • lets us roll all of the dice in an array, which we’ll do at the start of each round
  • parameters: an integer array representing five dice
  • return type: none

This function must set each element of the array to a random number from 1 to 6. Each random number must be generated by calling your rollDie function.

rollSome

  • lets us roll some of the dice, which is used during the reroll phase before scoring
  • parameters: an integer array representing five dice and a string containing the labels of any dice to reroll
  • return type: none

Each die has a label from a to e (first die is a, second is b, etc.). The function’s second parameter may contain any combination of these letters in any order. If a die’s label appears in that string, then that die must be rerolled. If a die’s label does not appear in that string, then that die must not be changed.

The dice are stored in an array, which means each die has an index starting at 0. You can convert a die’s index to its label with the formula (char)('a' + index). You can check whether a character x appears in a string y by calling its indexOf method: y.indexOf(x). If x appears in the string, then this will evaluate to a valid index (an integer greater than or equal to zero). If x does not appear in the string, then this will evaluate to negative one.

You can use a counting loop to check each die’s index and reroll the die if its label appears in the string. Here’s an outline of the whole function:

rollSome(dice, labels):
    int index = 0
    while index < number of dice
        char label = convert index to label
        if labels.indexOf(label) isn't negative one
            dice[index] = roll a new die
        increase index by one

2. Scoring Functions (5 points)

Next we’ll write some functions to score the user’s dice. We’ll leave out the straight and full house categories (these are the hardest to score), but you’re free to include them if you want to spend extra time on your dice game.

countDice

  • tells us how many times a die value appears in our array, which is important for most of the score calculations
  • parameters: an integer array representing five dice and an integer representing the die value we want to count
  • return type: integer

You should make a sum variable, then check whether each element of the array is equal to the value we’re counting. If it is, then increase the sum by 1. With a counting loop this is easy:

countDice(dice, value):
    sum = 0
    index = 0
    while index < number of dice
        if dice[index] = value
            increase sum
        increase index
    return sum

countMostCommon

  • tells us how many times the most common value appears in our array, which is useful for scoring 3 of a kind, 4 of a kind, and 5 of a kind (yahtzee)
  • parameters: an integer array representing five dice
  • return type: integer

This function should use a counting loop, but you’re not counting indexes in the array. Instead, you’re counting from 1 to 6 and checking how many times each value appears in the array. Your countDice function can take care of searching the array, so all you need to do is keep track of the highest count you’ve seen so far:

countMostCommon(dice):
    count = 0
    dieValue = 1
    while dieValue <= 6
        count = max(count, countDice(dice, dieValue))
        increase dieValue
    return count

sumDice

  • adds all the die values
  • parameters: an integer array representing five dice
  • return type: integer

You just need to return the sum of the numbers in an array. You can use a simplified version of the loop in countDice to do this.

calculateScore

  • tells us the score for our dice in one category
  • parameters: an integer array representing five dice and an integer representing the score category
  • return type: integer

This is where most of the work happens, but the other three functions will make this one a lot easier to write. There are ten score categories you need to handle, but six of them can be scored with the same code, so you really just have five cases. You may assume that the category is always a valid number or return 0 for invalid categories.

Here are the categories and their rules:

  • 1, 2, 3, 4, 5, and 6: Aces, Twos, Threes, Fours, Fives, Sixes
    • score equals the sum of any dice matching the category number
    • for example, scoring category 5 with the dice 1, 5, 6, 5, 5 would result in 15 points, the sum of all dice showing a 5
  • 7: Three of a Kind
    • if there are at least three matching dice, then the score is the sum of all dice
    • if there are fewer than three matching dice, then the score is 0
  • 8: Four of a Kind
    • if there are at least four matching dice, then the score is the sum of all dice
    • if there are fewer than four matching dice, then the score is 0
  • 9: Five of a Kind (Yahtzee)
    • if there are at least five matching dice, then the score is 50
    • if there are fewer than five matching dice, then the score is 0
  • 10: Chance
    • the score is the sum of all dice

To get started, I recommend breaking up this function with some if statements:

calculateScore(dice, category):
    if category is 1 to 6
        ...
    else if category is 7
        ...
    else if category is 8
        ...
    else if category is 9
        ...
    else
        ...

Make sure you end in an else, which represents chance. We don’t check because we’re assuming a valid category number. If you don’t want to make that assumption, then add an additional case to catch any invalid categories:

    (earlier cases elided)

    else if category is 9
        ...
    else if category is 10
        ...
    else
        return 0

For categories 1 to 6: return the category multiplied by countDice(dice, category).

Categories 7, 8, and 9 should all use countMostCommon to check whether we have the required number of matching dice:

if countMostCommon(dice) > minimum (minimum depends on category)
    return ??? points
else
    return 0

If you have the required number of matching dice, then you either return sumDice(dice) (three/four of a kind) or 50 (five of a kind). If not, then you return 0.

Chance is the easiest: just return sumDice(dice).

printScorecard

  • displays the user’s score at the end of the game
  • parameters: an integer array representing the user’s score in each of the ten categories
  • return type: none

This function must print the name and number of each score category followed by the score stored in the array (if the category was used) or a hyphen (if still available). Remember that the category numbers start at 1, but array indexes start at 0. The score for category 1, Aces, is stored at index 0.

After printing every category, print the user’s total score. Remember to skip any available categories when calculating the total score, or you’ll get the wrong number. Here’s some pseudocode to help:

printScorecard(scores):
    totalScore = 0
    index = 0
    while index < number of categories
        number = index + 1
        name = categories[index]
        score = scores[index]
        if category is available
            print "number. name: -"
        else
            print "number. name: scores[index]"
            totalScore = totalScore + scores[index]
        increase index
    print "Total: totalScore"

printCategories(scores, dice):

5. Reroll Command (5 points)

At the beginning of your round function, make a variable to track how many times the user is allowed to reroll their dice. Set the variable to 2 (normal Yahtzee rules allow up to two rerolls per round).

A reroll command consists of the letter “r”, followed by a space, followed by one or more letters from “a” to “e” to indicate which dice to reroll. If the user enters a command starting with r, then assume it is a properly-formatted reroll command.

To implement the reroll command:

  • if the user has any remaining rerolls
    • call rollSome and pass it the entire command string (extra characters that aren’t letters from a to e should have no effect on the rollSome function’s behavior)
    • decrease the reroll variable by 1
  • if the user does not have any remaining rerolls
    • print a message telling them they can’t reroll any more dice

To summarize the changes, here’s updated pseudocode for round:

round(scores):
    rerolls = 2
    ...
    while not done
        ...
        if command starts with "r"
            if rerolls > 0
                rollSome(dice, command)
                decrease rerolls
            else
                print warning message
        ...

6. Score Command (5 points)

A score command consists of the letter “s”, followed by a space, followed by a number from 1 to 10 to indicate which score category the user wants to claim this round. If the user enters a command starting with s, then assume it is a properly-formatted score command.

To implement the score command:

  • take a substring from the command starting at index 2 and parse it as an integer to get the category number
  • subtract one from the category number to get the category index
  • if the category number is not between 1 and 10 (inclusive)
    • print a message telling the user they must choose a category from 1 to 10
  • if the score at the category index is not -1
    • print a message telling the user they’ve already used that category
  • otherwise
    • calculate the score for that category
    • assign the score to the scores array at the category index
    • print a message confirming the name of the category and how many points were scored
    • set done to true so the round can end
    • use IO.readln to prompt the user to press enter to continue, which gives them a chance to see the score confirmation before the next round starts

round(scores):

            substring = command.substring(2)

                print warning message
            else if category isn't available
                print warning message
